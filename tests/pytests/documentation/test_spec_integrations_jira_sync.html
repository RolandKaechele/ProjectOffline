integrations/jira_sync.py — Jira → Project (run_sync)

61 tests
Jira → Project sync execution engine. Fetches Jira issues matching a configured filter and creates or updates MPXJ tasks in the in-memory project. Epics become summary+rollup container tasks; sub-tickets are wired as MPXJ child tasks via setParentTask. Per-task Jira metadata is persisted in a task_jira key in the .custom-props.json sidecar. All MPXJ/Java calls and Jira API interactions are mocked — no JVM or real Jira server required. Covers sidecar I/O, helper functions (_issue_type_label, _status_to_percent, _build_jira_key_lookup), and the full run_sync flow including early-exit paths, task creation, task update, epic hierarchy, result counters, [CREATE]/[UPDATE] debug log tokens, task name = bare summary (no bracket prefix), and paginated issue fetching (page count, startAt, field list, empty page).
