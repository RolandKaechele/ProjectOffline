settings_dialogs.py

161 tests
Settings dialog classes: KeePass config/new-entry, Jira server edit/list/sync-config — modal flags, titles, data loading. JiraSyncConfigDialog: server dropdown (disabled when empty, enabled with servers, pre-selects saved server), Add button for server management; filter and field-checkbox state is persisted inside a single jira2project JSON container in custom properties (backward-compatible load from legacy flat keys). ConfluenceCalendarConfigDialog: auth combo, KeePass stack pages, calendar props, timezone QComboBox (editable, IANA data via tzdata, QCompleter with MatchContains + case-insensitive, NoInsert policy, defaults to Europe/Berlin, pre-selects via index so popup scrolls to selection, unknown timezone kept as free text, saves on accept; timezone tests use an autouse monkeypatch fixture to inject a known zone list so results are independent of tzdata installation). EmailServerEditDialog: SMTP-only account editor (no KeePass section), “From KeePass…” button looks up sender address via AD/LDAP using the saved KeePass username. EmailConfigDialog: project-aware dialog with account QComboBox, “Manage Accounts…” button, per-project sender display-name field; active account and sender name are stored in the project sidecar JSON; KeePass entry remains in QSettings.
