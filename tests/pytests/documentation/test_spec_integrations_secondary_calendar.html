integrations/secondary_calendar_integration.py

41 tests
Per-resource secondary holiday calendar assignments persisted in project custom properties. Covers JSON map parsing and validation (_parse_map), reading and writing the full calendar map (get_secondary_calendar_map, set_secondary_calendar_for_resource including set/clear/noop paths), resolving a calendar object for a resource (resolve_secondary_calendar by name and UID, not-found path), inferring a secondary calendar from AD user metadata (infer_secondary_calendar_from_ad: state/country matching, scoring, auto-match tag), and the full assign-and-persist flow (assign_secondary_calendar_from_ad). Also covers the parent_dates named-only filter: anonymous (name=null/empty) default-calendar exceptions must not be added to parent_dates so that secondary-calendar holidays sharing a start date with a Schulferien block are not incorrectly suppressed in the Team Planner.
