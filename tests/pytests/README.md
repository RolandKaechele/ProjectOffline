## Requirements

Install test dependencies alongside the application dependencies:

```bash
pip install pytest
pip install -r requirements.txt
```

> **Java / MPXJ:** No JVM is needed to run the tests. All Java classes (`org.mpxj.*`,
> `jpype`, `java.time`) are replaced with `MagicMock` objects via `sys.modules` patching
> or `unittest.mock.patch`.
>
> **Confluence dependencies:** `requests` and `playwright` are stubbed in `sys.modules`
> before `integrations.confluence_calendar_integration` is imported, so the Confluence Calendar tests run fully
> offline without a browser or a Confluence server.

## Running the tests

Run the full suite from the project root (path configured in `pyproject.toml`):

```bash
pytest
```

Run only a specific module:

```bash
pytest tests/pytests/test_history_manager.py -v
pytest tests/pytests/test_gantt_view.py -v
```

Run only tests matching a keyword:

```bash
pytest tests/pytests/ -v -k "undo"
pytest tests/pytests/ -v -k "load_project"
```

Show a short summary of failures only:

```bash
pytest tests/pytests/ -q
```

## Qt (headless)

All widget tests run headlessly. The environment variable `QT_QPA_PLATFORM=offscreen`
is set automatically by `conftest.py` before any `QApplication` is created, so no
display server is required.

## conftest.py â€” shared fixtures

`conftest.py` is loaded automatically by pytest before any test module.  It:

1. **Stubs Java / JVM modules** â€” injects `MagicMock` objects for `jpype`, `mpxj`,
   `java.time`, `org.mpxj.*`, `pykeepass`, `jira`, `requests`, etc. into `sys.modules`
   so that lazy `import jpype` / `import java.time` calls inside source files succeed
   without a real JVM being installed.  Individual tests can still override specific
   attributes with `patch.object(jpype, 'isJVMStarted', ...)`.

2. **Prepends `src/` and `src/views/` to `sys.path`** so every source module is
   importable without installation.

3. **Sets `QT_QPA_PLATFORM=offscreen`** for headless widget tests.

4. **Provides session fixtures and mock factories:**

| Name | Scope | Description |
| - | - | - |
| `qapp` | session | Single `QApplication` shared across the entire test session |
| `mock_task` | function | `MagicMock` shaped like an MPXJ `Task` (via `make_mock_task`) |
| `mock_resource` | function | `MagicMock` shaped like an MPXJ `Resource` (via `make_mock_resource`) |
| `mock_project` | function | `MagicMock` shaped like an MPXJ `ProjectFile` (via `make_mock_project`) |

Factory functions `make_mock_task`, `make_mock_resource`, `make_mock_project`,
`make_mock_ldt` are importable directly from `conftest` for custom configurations.
