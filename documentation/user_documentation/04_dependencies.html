Dependencies

Create and manage task predecessor relationships — Finish-to-Start, Finish-to-Finish, Start-to-Start, and Start-to-Finish.

Home› Dependencies

The Dependency View

Open the Dependencies tab to see every predecessor relationship in the project. Each row represents one link between two tasks.

ColumnDescription
Task IDID of the successor task (the task that depends on another)
Task NameName of the successor task
Predecessor IDID of the task that must progress first
Predecessor NameName of the predecessor task
Link TypeRelationship type: FS, FF, SS, or SF
LagAdditional delay (positive) or overlap (negative) e.g. 2.0d

Relationship Types

CodeNameMeaning
FSFinish-to-StartSuccessor starts after predecessor finishes. Most common type.
FFFinish-to-FinishSuccessor finishes no earlier than predecessor finishes.
SSStart-to-StartSuccessor starts no earlier than predecessor starts.
SFStart-to-FinishSuccessor finishes no earlier than predecessor starts.

Adding a Dependency

  1. In the Dependencies tab, right-click → Insert Dependency (or press Insert).
  2. The Dependency dialog opens — select the successor task and the predecessor task from the drop-down lists.
  3. Choose the Link Type and enter an optional Lag (days).
  4. Click OK. The relationship is stored and the Gantt chart automatically redraws.

Editing a Dependency

Double-click any row in the Dependencies tab to re-open the dialog and change the link type or lag. Predecessor and successor tasks cannot be changed in-place — delete the row and create a new one instead.

Deleting a Dependency

Select the row and press Delete, or right-click → Delete Dependency. Use Undo to reverse.

Viewing in the Gantt Chart

Dependency arrows are not drawn on the Gantt chart in the current release. The Predecessors column in the task sheet shows each predecessor as ID — Task Name (e.g. 6 — Component design). Multiple predecessors are separated by commas.
