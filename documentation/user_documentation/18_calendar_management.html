Holiday Calendar Management

Automatically populate project calendars with public holidays for Germany (national + all 15 federal states) and 5 international countries — no internet connection required.

Home› Holiday Calendar Management

Overview

Project Offline can automatically create and fill project calendars with public-holiday dates so that the Gantt chart, Team Planner, and Resource Usage Graph correctly skip non-working days during scheduling. Holidays are computed entirely offline — no network connection, no external files, and no licence needed.

Holiday calendars are populated for a rolling 15-year window (current year −2 through current year +13).

No internet required All holiday dates are built into the application. Movable dates such as Easter, Corpus Christi, and Buß-und-Bettag are calculated algorithmically; fixed-date holidays use the official legal rules for each country.

Step 1 — Create a new project

Use File → New (Ctrl+N). The application immediately:

  1. Reads the OS locale monetary settings and applies them as the project currency (symbol, currency code, decimal digits, symbol position). On a German Windows system this sets € / EUR automatically instead of the generic $ default.
  2. Creates a project calendar named Standard (Deutschland).
  3. Populates it with all 9 German national public holidays for the 15-year window.
  4. Opens the Select Additional Calendars dialog (see below).
  5. After you close that dialog, opens the Project Information dialog so you can set the start and end dates.

Step 2 — Pick optional regional / country calendars

The Select Additional Calendars dialog opens automatically after File → New. It lists:

SectionEntriesWhat is added
German Federal States All 15 states State-specific extra holidays (Corpus Christi, Epiphany, Assumption, All Saints', Buß-und-Bettag, Children's Day, etc.)
Other Countries France, India, Romania, China, Japan National public holidays for the selected country

Tick the calendars you need and click OK. Each ticked entry creates a separate named project calendar pre-loaded with the corresponding holidays. Unticked entries are simply skipped — you can add them later via Add Holidays… (see below).

Tip Select your country's state calendar to ensure regional bank holidays (e.g. Corpus Christi in Bavaria, or Reformation Day in Brandenburg) are treated as non-working days in your schedule.

Adding or changing calendars in an existing project

You can manage holiday calendars at any time via the Project Information dialog:

  1. Open Project → Project Information… (or press Alt+F11).
  2. Go to the Calendars tab.
  3. Select the calendar you want to modify in the list on the left.
  4. Use the action buttons on the right:
ButtonAction
Import ICS… Opens a file-picker dialog. Select any .ics or .ical iCalendar file (e.g. a corporate public-holiday feed, a shared team out-of-office calendar). Every event in the file is converted to a non-working exception on the selected calendar.

Multi-day events are expanded day-by-day — a conference that runs Monday to Friday generates five individual non-working exception days.
Add Holidays… Re-opens the Select Additional Calendars dialog. Tick any German state or country you want to add. The newly selected calendars are created immediately; already-existing calendars are not duplicated.
ICS import — what is imported The importer reads the DTSTART value of each VEVENT block and marks that date as non-working. Events that span multiple days (DTSTART + DTEND) generate one non-working day per day in the range. VTIMEZONE, recurrence rules (RRULE), and alarms (VALARM) are ignored — only the explicit date is used.

How holiday calendars affect scheduling

Holiday calendars in Project Offline work the same as in Microsoft Project:

  • Non-working exception days are skipped when calculating task durations and dependency dates.
  • Tasks that would land on a holiday are shifted forward automatically.
  • The Gantt chart and Team Planner shade holiday columns distinctly from regular weekends.

To assign a specific calendar to a task or resource, use the Calendar drop-down in the Task Information or Resource Information dialog respectively.

Base Calendar Selector

In Project Information → Calendars, the Base Calendar field is an editable selector. The project default calendar appears first with a star prefix, the currently edited calendar is excluded, and the selected parent calendar is saved when you confirm the dialog.
