Baseline Tracking

Capture scheduled dates as a baseline snapshot, then track variances as the project progresses.

Home› Baseline Tracking

What Is a Baseline?

A baseline is a saved snapshot of the project schedule at a specific point in time (typically when the project is approved and work starts). As the project progresses you can compare current dates to the baseline to measure schedule variance.

Project Offline supports 11 independent baseline slots (Baseline 0 through Baseline 10), matching Microsoft Project's MSPDI format.

Setting a Baseline

  1. Open Project → Set Baseline… from the menu bar, or click the Set Baseline split-button in the BASELINE ribbon tab.
  2. Choose which slot to save into (Baseline 0 is the default).
  3. Click Set Baseline. The current Start, Finish, and Duration of every task are copied into the chosen slot, and the capture date is recorded in the project properties.

The Set Baseline ribbon button is a split-button — click the ▼ arrow for two additional quick-access modes:

  • Set into next free slot — automatically finds the lowest-numbered unoccupied slot (BL-0 through BL-10) and snapshots instantly, without showing the dialog.
  • Set all slots (bulk) — snapshots the current schedule into all 11 slots simultaneously.

Clearing a Baseline

Open Project → Clear Baseline…, select the slot, and confirm. All baseline fields for that slot are set to null and the capture date is removed from project properties.

The Clear Baseline ribbon button is also a split-button. Click the ▼ arrow and choose Clear all baselines to remove all data from all 11 slots simultaneously without navigating the dialog.

Baseline View

Switch to the Baseline tab to see the comparison table. Use the Reference and Compare to selectors to choose which slots to compare.

ColumnDescription
ID / NameTask identifier and name
BL Start / BL Finish / BL DurationValues stored in the reference baseline slot
Current Start / Current Finish / Current DurationCurrent schedule (or a second baseline when comparing two baselines)
Start Δ (d) / Finish Δ (d)Variance in calendar days (positive = late)

Cells in the Current Start/Finish/Duration columns are highlighted yellow whenever their value differs from the reference baseline cell.

Baseline Strip on Gantt Chart

When a baseline is set a thin dark strip appears below each task bar on the Gantt chart, showing the planned start and finish. The strip is drawn even when the Baseline tab is not active.

Comparing two baselines Set Compare to to any slot other than “Current” to compare two historical snapshots — for example, Baseline 1 (initial plan) vs Baseline 2 (re-plan after first month).

Quick-Access Split-Button Options

The Set Baseline and Clear Baseline buttons in the BASELINE ribbon tab are split-buttons. Clicking the main button area opens the normal dialog. Clicking the small ▼ drop-down arrow gives quick access to additional options without opening a dialog:

ButtonDrop-down optionEffect
Set Baseline ▼ Set into next free slot Automatically finds the lowest unoccupied baseline slot (BL-0 through BL-10) and snapshots the current schedule into it immediately, without showing the dialog.
Set all slots (bulk) Snapshots the current schedule into all 11 slots (BL-0 through BL-10) simultaneously in one step.
Clear Baseline ▼ Clear all baselines Removes all baseline data from all 11 slots simultaneously, without showing the dialog. Use with caution.
Tip: Use Set into next free slot when you want to capture intermediate progress checkpoints without having to count which slot numbers are already in use.
