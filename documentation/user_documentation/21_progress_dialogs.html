Progress Dialogs

Long-running operations in Project Offline run in the background and display a modal progress dialog so the application stays responsive.

Home› Progress Dialogs

Overview

Whenever Project Offline starts an operation that may take more than a second — such as opening a large project file, running a Jira sync, or exporting an SVG — a modal progress dialog appears automatically. The dialog:

  • Keeps the main window responsive (it can be repainted, resized, and receives keyboard shortcuts).
  • Shows either a percentage bar (for operations that report incremental steps) or a pulsing indeterminate bar (for operations whose total work cannot be known in advance).
  • Displays a status message below the bar describing what is happening.
  • Optionally includes a Cancel button for operations that support cancellation.

Cancelling an Operation

When a Cancel button is shown, clicking it sets a cancellation flag in the background thread. The operation will stop at the next safe checkpoint — it does not kill the thread forcefully. Any partial results (e.g. already-created tasks during a partial Jira sync) may still be applied to the project.

Non-cancellable operations Operations such as File Open and File Save do not show a Cancel button because interrupting them mid-way could leave the project in an inconsistent state.

Operation Timing Log (Debug Mode)

When Project Offline is started with the --debug flag, a timing log is included in the debug dump under the long_running_timings key. The log records the last 50 completed operations with their name, elapsed time in seconds, outcome (success / failure), and timestamp.

Example entry in the debug JSON dump:

[
  {
    "operation":       "Jira Sync",
    "elapsed_seconds": 4.231,
    "success":         true,
    "timestamp":       "2026-05-14T10:33:07"
  },
  {
    "operation":       "File Open",
    "elapsed_seconds": 1.087,
    "success":         true,
    "timestamp":       "2026-05-14T10:32:54"
  }
]

The timing log is useful for diagnosing performance issues — for example, identifying which sync operation is consistently slow on a given network.

Generating a debug dump With --debug enabled, use Help → Debug Dump to write a JSON snapshot that includes the timing log.
