Task Splitting & Merging

Interrupt a task mid-execution and resume it later — or merge the segments back into a single bar.

Home› Task Splitting & Merging

What Is Task Splitting?

Task splitting lets you model an interruption: the task stops on a given date and resumes at a later date. On the Gantt chart, splits appear as two (or more) bars connected by a dashed line. The total duration is preserved — only the continuous working time is redistributed.

Splitting a Task

  1. In the Gantt Chart or Task Sheet tab, right-click the task you want to split.
  2. Choose Split Task… from the context menu. (The option is only shown for non-summary, non-milestone tasks.)
  3. In the Split Task dialog, enter the split date — the day the task will pause.
  4. Optionally adjust the resume date — the day the task restarts after the gap.
  5. Click OK. The Gantt chart redraws with a dashed connector between the two segments.

How Splits Are Stored

Split information is stored in a companion .splits.json file saved alongside the project XML. This ensures the split data survives round-trips through MPXJ without loss. When you open a project, the application automatically looks for a matching .splits.json file.

Gantt Chart Rendering

ElementAppearance
First segmentNormal task bar up to the split date
GapDashed horizontal connector line
Subsequent segmentsNormal task bar from the resume date
Vacation overlaySemi-transparent orange block if a resource is on leave during the gap

Merging Segments

  1. Right-click the split task in the Gantt Chart or Task Sheet.
  2. Choose Merge Task Segments. (This option is only visible when the task has existing splits.)
  3. Confirm the merge. All split points are removed and the task becomes a single continuous bar again, spanning the original start to the last finish.
Multiple splits A task can have more than two segments. Repeat the Split Task action on an already-split task to add a second interruption. Merge Task Segments always removes all splits at once.
Summary tasks Summary tasks and milestones cannot be split. The context menu item is hidden for those task types.
