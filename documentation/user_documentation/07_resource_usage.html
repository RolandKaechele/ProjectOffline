Resource Usage Graph

Analyse daily, weekly, monthly and hourly workload for every resource in a colour-coded hour grid.

Home› Resource Usage Graph

What the View Shows

The Resource Usage Graph tab displays how many hours each resource is scheduled to work during every time period. Resources can be expanded to show the individual task contributions. The layout mirrors Microsoft Project's built-in Resource Usage view.

Zoom Modes

Day-width rangeModeColumn unit
4–6 pxMonthly1 calendar month — shows monthly total hours
7–13 pxWeekly1 ISO week (Mon–Sun) — shows weekly total hours
14–59 pxDaily1 calendar day — shows per-day hours
60–80 pxHourly1 work hour — skips weekends, uses project calendar hours

Use the global zoom slider (shared with the Gantt chart) to switch modes.

Reading the Grid

Each cell shows the number of hours a resource (or one of its task assignments) is scheduled in that period. Cells are coloured by utilisation level:

ColourMeaning
White / defaultWithin Max Units — normal workload
Light orangeApproaching the maximum (configurable threshold)
RedOver-allocated — hours exceed the resource's Max Units
Light greyNon-working day (weekend or public holiday)

Collapsing Resource Rows

Click the ▼ / ▶ triangle next to a resource name in the left pane to collapse or expand its task sub-rows. Collapsed rows show only the resource total; expanding reveals each task assignment on its own line.

Hourly Mode Details

In hourly mode, the calendar header switches from day numbers to per-hour slots (e.g. 08:00, 09:00 …). Working hours are read from the project's default calendar — so a 7-hour workday will show 7 hour columns per working day. Saturdays and Sundays are skipped entirely.

Navigation The ◀ Today button scrolls the view to the current date. The frozen left pane (resource names + total work column) stays fixed while the hourly grid scrolls.
