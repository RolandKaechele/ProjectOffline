Email Integration

Send project exports (SVG Gantt charts, Team Planner snapshots, resource graphs) as email attachments directly from the application via SMTP.

Home› Email Integration

What Is Email Integration?

Project Offline includes built-in SMTP email support that lets you send project exports as email attachments without leaving the application. Email credentials are securely retrieved from your KeePass database, so no passwords are stored in application settings.

Secure by Design SMTP passwords are retrieved from KeePass at send-time only. The application stores only the SMTP server address and port—never passwords. SMTP account settings (server, port, TLS) are saved in email_configs.json next to the executable; the KeePass entry name stays in the Windows registry. Sender address and Sender Name are saved per-project in the project sidecar JSON (not in email_configs.json), so different projects can send from different addresses. The JSON file can be copied between machines to share your account list; it is excluded from version control.

Prerequisites

RequirementDetails
KeePass Database A configured and unlocked KeePass database with an entry containing your SMTP username and password. See KeePass Credentials for setup.
SMTP Server Access Access to an SMTP server (corporate mail server or email provider like Gmail, Outlook). Port 587 with STARTTLS is recommended.
SMTP Credentials Valid username and password for SMTP authentication, stored in your KeePass database.

Step 1 — Configure Email Settings

Open the Email Configuration dialog from the Report ribbon tab by clicking Email Config. The dialog is always available (not project-gated) since the KeePass entry is a global setting. When a project is open, the active account, sender address, and sender display name are stored per-project in the project sidecar JSON.

SettingDescription
KeePass Entry Select the KeePass entry containing your SMTP username and password. Shared by all SMTP accounts. Only available when the KeePass database is unlocked.
Account Select which named SMTP account to use for this project. Click Manage Accounts… to add, edit, or delete accounts.
Sender Address The SMTP From: address for this project (e.g. alice@corp.com). Saved per-project in the sidecar JSON. Click From KeePass… to auto-fill by looking up the KeePass entry username in Active Directory.
Sender Name Optional display name shown in the From: header (e.g. Alice Smith). The email is sent as "Alice Smith <alice@corp.com>" when set. Saved per-project in the sidecar JSON — different projects can have different sender names. Leave blank to send from the plain address.
Corporate Mail Servers: use “From KeePass…” in Email Config Many corporate SMTP servers require the sender address to match the authenticated username. If you receive a “5.7.60 Client does not have permissions to send as this sender” error, open Report → Email Config and click From KeePass… next to the Sender Address field to auto-fill your email from Active Directory.

Step 2 — Test Your Connection

Before sending emails, verify your SMTP configuration by clicking Test Connection. This validates the server connection and authentication without sending any email.

If the test fails, check:

  • SMTP server hostname and port are correct
  • KeePass database is unlocked and the selected entry contains valid credentials
  • Your network allows outbound connections to the SMTP port
  • The SMTP server supports STARTTLS if enabled

Step 3 — Send a Test Email

After a successful connection test, click Send Test Email to verify end-to-end email delivery. You'll be prompted to enter a recipient address.

Tip — Self-Test Click Use My Email to automatically fill your own email address from Active Directory. This lets you verify email delivery by sending to yourself.

Managing Multiple Email Accounts

Project Offline supports multiple named SMTP configurations — for example, a corporate mail server and a personal account. Click Email Accounts in the Report ribbon tab to open the account manager.

ButtonDescription
Add…Open the account editor to create a new named SMTP configuration.
Edit…Open the account editor for the selected account.
DeleteRemove the selected account permanently.
Set ActiveMark the selected account as the one used for all email operations.
Move Up / Move DownReorder accounts in the list.
Active Account The account marked with ★ is used when sending emails. To switch accounts, select the desired account and click Set Active, then Save.

Adding or Editing an Email Account

Click Add… or Edit… to open the account editor. Each account stores only its SMTP server settings (Name, Server, Port, TLS). Sender address is stored per-project in Email Config. KeePass credentials are configured separately via Email Config and apply globally to all accounts.

Sender address and credentials are per-project All email accounts share the KeePass entry configured in Report → Email Config. The sender address is also set there, per-project, in the “Sender (Per-Project)” section — not in the account editor. Use From KeePass… to auto-fill it from Active Directory.
Legacy Migration If you previously configured a single email account using Email Config, it is automatically migrated to the accounts list as an entry named “Default” the first time you open Email Accounts. No manual migration is required.

Sending Project Exports

Once email is configured, you can send project exports as email attachments. The following export formats are typically sent via email:

  • Gantt Chart (SVG) — Complete project Gantt chart as scalable vector graphics
  • Team Planner Snapshot (SVG) — Current Team Planner view as SVG
  • Resource Utilisation Graph (SVG) — Resource usage over time

To send an export:

  1. Generate the export file using the appropriate export function (e.g., File ? Export ? Gantt Chart (SVG))
  2. Use the email integration API to attach the file and send it to recipients
Note — Attachments The email integration supports multiple attachments. Files are sent as MIME attachments with appropriate content types (e.g., image/svg+xml for SVG files).

When Email Is Not Yet Configured

If you have not yet added an SMTP account, or have not selected an active account via Email Config, the three email export actions in the Report ribbon are automatically greyed out (disabled):

  • Email Gantt SVG — in the Gantt Export split-button popup
  • Email to All Resources — in the Gantt Export and Resource SVG popups
  • Email Timeline SVG — in the Timeline Export split-button popup

Hovering over a greyed-out action shows a tooltip:

No email account configured.
Use Email Accounts in the Report ribbon to add an SMTP account,
then select it via Email Config.

The enabled/disabled state is evaluated automatically:

  • When the application starts
  • When the Email Accounts dialog closes
  • When the Email Config dialog closes
Tip — Getting Started If the email export actions appear greyed out, open Report → Email Accounts to add your SMTP account, then open Report → Email Config and select the account you just added. The email actions will become active immediately when the dialog closes.

Security Best Practices

PracticeDetails
Use KeePass for Credentials Never enter SMTP passwords directly into the application. Always use KeePass to store and retrieve credentials securely.
Enable STARTTLS Always enable Use STARTTLS when using port 587 to encrypt the connection between the application and the SMTP server.
Verify Sender Address Ensure your sender address matches your authenticated username to avoid "send as" permission errors.
Lock KeePass When Not in Use Lock your KeePass database when you're away to prevent unauthorized access to SMTP credentials.
Security Warning Email credentials are never written to the Windows registry, log files, or debug dumps. The application stores only the SMTP server configuration and KeePass entry path. Passwords are retrieved from KeePass at send-time only.

Troubleshooting

ProblemSolution
"Email integration not fully configured" Ensure all required fields are filled: SMTP server, sender address, and KeePass entry. Click Save in the Email Configuration dialog.
"Failed to retrieve SMTP credentials" Unlock your KeePass database and ensure the selected entry contains both username and password fields.
"SMTP authentication failed" Verify the KeePass entry contains the correct username and password. Test with your email client to confirm the credentials work.
"Client does not have permissions to send as this sender" Your sender address must match your authenticated username. Open Report → Email Config and click From KeePass… in the Sender section to auto-fill your email from Active Directory.
Connection timeout Check your network connectivity and firewall settings. Ensure outbound connections to the SMTP port (587) are allowed.
Tip — Debug Logging Run the application with the --debug flag to enable detailed logging. The debug dump will include email configuration details (without passwords) to help diagnose issues.
