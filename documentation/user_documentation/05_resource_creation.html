Resource Creation & Editing

Add team members and material resources, set availability and pay rates, and manage the resource sheet.

Home› Resource Creation & Editing

Resource Sheet Columns

#ColumnDescription
0IDSequence number
1NameResource display name (used in task assignments and Team Planner)
2TypeMPXJ resource type: WORK, MATERIAL, or COST
3Max UnitsMaximum availability as a percentage (visual progress bar; >100% = red)
4Standard RateRegular pay rate
5Overtime RateOvertime pay rate

Adding Resources

  • Right-click in the resource sheet → Insert Resource
  • Press Insert while the resource sheet has focus
  • Toolbar Add Resource button on the Resources tab

Resource Information Dialog

Double-click any resource row to open the Resource Information dialog.

The top of the dialog shows a 64 px circular avatar: if an AD photo is stored for the resource it is displayed; otherwise a type-specific icon is shown ( 👷 Work, 📦 Material, 💰 Cost). The avatar refreshes automatically when you change the Type field.

FieldNotes
NameShown in task assignments and the Team Planner. Must be unique — a warning is shown if another resource with the same name already exists.
TypeWork (person or equipment), Material (consumable), Cost (fixed cost). Changing the type also updates the avatar icon.
EmailE-mail address. Visible only for Work resources. Pre-populated from MPXJ. Use Look up in AD… to fill it from the corporate directory.
DepartmentDepartment or organisational unit. Visible only for Work resources. Filled automatically when using Look up in AD… or AD sync.
Max UnitsEnter as a percentage (e.g. 100 = full time, 50 = half time).
Standard RateHourly or daily rate for cost calculations.
Overtime RateRate applied to overtime hours.
NotesFree-text notes about this resource.
AD photo persistence When a resource photo is retrieved via Look up in AD… it is stored in a <project>.thumbnails.json sidecar file alongside the project XML. Photos and department values survive close and re-open; the sidecar is pruned automatically when a resource is deleted.
Max Units normalisation MPXJ stores Max Units as a fraction (1.0 = 100%) in XML files but as a percentage (100.0) in binary MPP files. Project Offline normalises these automatically so the display always shows a percentage. Values are saved back as fractions so files round-trip correctly.

Deleting Resources

Select the row and press Delete, or right-click → Delete Resource. Deleting a resource that is assigned to tasks removes those assignments automatically. If the resource has a personal calendar, that calendar is also removed from the project calendar list automatically — no orphaned calendars accumulate. Use Undo to reverse.

Resource Sheet Hover Tooltip

Hovering over any row in the resource sheet shows a rich tooltip containing:

  • A 64 px circular avatar (AD photo if stored, otherwise the type icon)
  • Resource name and type
  • Department (if set)

The tooltip rebuilds only when you move to a different row, keeping it responsive.

Active Directory Sync

If Settings → Active Directory is enabled, a Sync from AD button appears on the Resources ribbon tab. It looks up each resource by display name and fills in the email and department fields from the corporate directory. See Active Directory Sync for details.

Individual resources can also be looked up without leaving the Resource Information dialog: press Look up in AD… next to the E-Mail field to search Active Directory by the resource’s current display name. A single match pre-fills the E-Mail and Department fields; multiple matches open a selection list. You can still edit the fields before clicking OK. See Manual Lookups for details.

Automatic UID assignment Every new resource receives a unique integer UID automatically — the smallest positive integer not already assigned to any other project resource. This ensures the resource appears immediately in the Team Planner and that task assignment rows always display the correct resource name rather than “(unknown)”.
