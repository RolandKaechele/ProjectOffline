Confluence Calendar Sync

Automatically import public holidays and approved leave from Confluence Team Calendars and apply them as non-working days to your project schedule.

Home› Confluence Calendar Sync

What Does Calendar Sync Do?

The calendar sync feature connects to your organisation's Confluence Team Calendars (via SSO / browser-based authentication) and pulls leave events and public holidays for the configured space. These are then applied as non-working exceptions to:

  • The project calendar (public holidays affect all tasks)
  • Individual resource calendars (leave affects only the relevant resource)

Once applied, the Gantt chart and Team Planner automatically account for those non-working days in scheduling calculations.

Prerequisites

  • The project must be saved as a .mpp or .xml file first.
  • The CALENDAR custom fields must be configured in the project properties (see below).
  • Access to the Confluence server (network connectivity and SSO credentials).
  • Optional: the playwright Python package installed for SSO browser support.

Step 1 — Configure Confluence Settings

The easiest way to set up the Confluence connection is the dedicated Confluence Calendar Configuration dialog. Open it from:

  • RESOURCE ribbon → Confluence group → Configure…
  • or Project → Configure Confluence Calendar…
Timezone autocomplete The Timezone field is an editable combo box. Type any part of the name — e.g. “berlin”, “paris” or “new_york” — and the list filters instantly. Matching is case-insensitive. Default: Europe/Berlin.

Alternative: configure via Project Information → Custom Fields

The same four keys can also be viewed and edited directly in the project file’s custom properties: Alt+F11 or Project → Project Information… → Custom Fields tab.

Required Custom Fields

Property NameValueRequired?
CALENDAR Base URLFull URL of your Confluence server, e.g. https://confluence.example.comRequired
CALENDAR Space KeyThe Confluence Space key to query (e.g. PROJ, IT)Required
CALENDAR TimezoneIANA timezone name (default: Europe/Berlin). Editable in the Configuration dialog with autocomplete.Optional
CALENDAR Days AheadNumber of days ahead to sync (default: 365, range 1–3650)Optional

Step 2 — Run the Sync

  1. Click the RESOURCE ribbon tab.
  2. Click Sync Calendar in the Confluence group.
  3. If this is the first sync (or the SSO session has expired), a Chromium browser window opens automatically for you to complete the SSO login. Close the browser once login is confirmed.
  4. The sync runs in the background. A progress message appears in the status bar.
  5. On completion, non-working exceptions (public holidays and leave events) are applied to the project and resource calendars.

Authentication (SSO)

Calendar sync uses Playwright to automate the SSO login flow:

  • First run / expired session: A Chromium browser window opens. Log in normally with your corporate SSO credentials. Once the browser shows the Confluence homepage, close the browser window. The session state is saved to ~/.confluence_playwright_state.json (owner-only file permissions) so future syncs are instant.
  • Valid saved session: The sync runs headlessly with no browser window.
Session file security The saved session file is stored with owner-only permissions (mode 600 on Linux/Mac, icacls restricted on Windows). Never share or commit this file.

KeePass Auto-fill for Microsoft SSO (Optional)

If your Confluence instance uses Microsoft Azure AD for SSO, you can configure a KeePass entry to pre-fill the email and password fields automatically. Only the MFA step (authenticator app, SMS code, etc.) needs to be completed manually.

Setup

  1. Open Settings → KeePass Configuration…
  2. Enter your master password and click Unlock.
  3. In the Confluence SSO Auto-fill section that appears, select the KeePass entry whose username and password match your corporate Microsoft login.
  4. Click OK to save.
How it works When the sync opens a browser for SSO login and detects the Microsoft login page (login.microsoftonline.com), it automatically fills the email and password fields from the selected KeePass entry. The browser then shows whatever MFA prompt your organisation requires — complete it as normal and the sync proceeds once Confluence is reached.

What Gets Synced

Calendar event typeSynced?Effect
Public holidaysYesAdded as non-working days to the project calendar; all tasks skip these days. Only single-day events are imported — multi-day spans such as school-holiday blocks (Schulferien) are skipped so they do not grey out entire weeks in the Team Planner.
Approved leave / vacationYesAdded as non-working exceptions to the matching resource calendar; also shown as orange overlay in Team Planner
Meetings / eventsNoNot imported (only leave-type and holiday-type events are relevant)

After the Sync

After a successful sync:

  • The Gantt chart bars are redrawn to skip the imported non-working days.
  • Resource leave appears as orange overlays on the Team Planner rows.
  • Task durations that cross non-working periods are automatically extended.
Re-run as needed Re-run the sync whenever the Confluence Team Calendar is updated (new leave approvals, changed holidays). The sync is non-destructive — it overwrites only calendar exceptions, not task dates or resource assignments.
