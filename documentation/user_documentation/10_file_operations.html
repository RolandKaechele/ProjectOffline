Open, Save & Export

Open project files in multiple Microsoft Project formats, save as MSPDI XML, and export the Gantt chart as an image.

Home› Open, Save & Export

Opening a Project

Use File → Open… (Ctrl+O) or the toolbar button. The file is read by the MPXJ library, which supports all Microsoft Project formats:

ExtensionFormatNotes
.mppMicrosoft Project binaryRead-only source format; the app always saves as XML
.xml / .mspdiMicrosoft Project XMLNative save format (MSPDI schema)
.mptMicrosoft Project template
.mpxMicrosoft Project ExchangeLegacy text-based format
MPP files When you open an .mpp file and then save, the file is written as MSPDI XML — not back to binary MPP format. Choose a new filename with an .xml extension to keep both files.

Saving

Use File → Save (Ctrl+S) or File → Save As… to write the current project to disk. Files are always saved in MSPDI XML format. Microsoft Project 2016+ can open MSPDI XML files directly.

Recent Files

File → Recent Files shows the last-opened projects. Select one to open it immediately without the file browser.

Exporting the Gantt Chart

Use File → Export Gantt Chart… to render the current Gantt chart view as a PNG image or PDF. The export dialog lets you choose:

OptionDescription
Date rangeExport all tasks, or a specific date window
FormatPNG (raster) or PDF (vector-friendly)
ScalePixel/day zoom — use higher values for print quality
Include baselineWhether to draw baseline strips on the exported bars
Exporting for presentations Set the zoom to 14 px/day or higher before exporting to get readable day labels in the calendar header. The Timeline strip can also be exported separately from File → Export Timeline….

Auto-Save

Project Offline writes a backup copy every few minutes to the system temp folder. If the application closes unexpectedly, the backup is offered for recovery on the next launch.

Sidecar Files

When a project is saved as XML, Project Offline writes a set of sidecar files alongside the .xml file to preserve data that the MPXJ MSPDIWriter does not persist in the standard schema:

Sidecar fileContents
<project>.xml.custom-props.json Per-project custom properties (Jira config, Confluence config, etc.)
<project>.xml.cal-exc-names.json Calendar exception names — MSPDI XML does not store exception names; this file restores them on the next open so holiday and vacation block labels are preserved. Removed automatically when no named exceptions remain.
<project>.splits.json Task split-segment data (start/end of each non-contiguous work segment)
<project>.timeline.json Tasks and milestones pinned to the Timeline strip
<project>.thumbnails.json AD profile photos and department strings (base64-encoded, per resource UID)
Keep sidecar files with the project When you copy or move a project file, copy all sidecar files with it. Without the sidecars, calendar exception names, task splits, timeline items, and AD photos will be lost on the next open.
