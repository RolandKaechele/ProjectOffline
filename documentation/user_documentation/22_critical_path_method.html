Critical Path Method (CPM)

Automatically identify the tasks that determine your project finish date and analyse schedule float for every task — entirely offline.

Home› Critical Path Method

What Is the Critical Path?

The critical path is the longest chain of dependent tasks in your project. Any delay to a task on this path directly delays the project finish date. Tasks not on the critical path have float (slack) — spare time that can absorb small delays without affecting the end date.

Project Offline runs a forward pass (earliest possible start/finish for every task) and a backward pass (latest allowable start/finish without delaying the project) to compute two float values per task:

ValueMeaning
Total FloatHow much the task can slip before the project is delayed. Tasks with Total Float ≤ 0 are critical.
Free FloatHow much the task can slip before it delays the next task (without affecting the project end).
Offline calculation — CPM runs entirely within Project Offline using your project’s existing task dates, durations, and dependency links. No server connection or additional licence is required.

Opening the CPM Results Panel

  1. Open a project file.
  2. Click the REPORT ribbon tab.
  3. In the Views group, click CPM Results.

The CPM Results tab appears at the right of the tab bar and becomes active. Alternatively, enable Show CPM Results panel in CPM Settings to make it permanently visible.

Reading the CPM Results Table

The table shows one row per task (summary tasks are included but computed from their leaf descendants). Rows are colour-coded:

ColourMeaning
RedCritical — Total Float ≤ threshold (default 0 days). Any delay here delays the project.
AmberNear-critical — small float; worth monitoring closely.
White / greyNon-critical — has comfortable float.

Columns

ColumnDescription
Task NameTask name from the project.
DurationTask duration in working days.
Early StartEarliest possible start date (forward pass).
Early FinishEarliest possible finish date (forward pass).
Late StartLatest allowable start without delaying the project (backward pass).
Late FinishLatest allowable finish without delaying the project (backward pass).
Total FloatLate Finish − Early Finish, in working days.
Free FloatSlack before the next successor task is delayed, in working days.
StatusCRITICAL, near-critical, or —

Filter and Export

  • Critical tasks only checkbox — hides non-critical rows so you can focus on blockers.
  • Export CSV… button — saves the currently visible rows (respecting the filter) as a UTF-8 CSV file for use in Excel or other tools.

The status bar at the bottom of the panel always shows the total count of critical and near-critical tasks.

Opening a Task from the CPM Results Panel

Double-click any row in the CPM Results table to open the Task Information dialog for that task. The dialog's Schedule tab will show the correct critical status and float values computed by the CPM engine.

CPM Settings

Open REPORT ribbon → CPM Settings to customise the analysis.

Calculation Settings (saved per project)

SettingDefaultDescription
Critical slack threshold 0 days Tasks with Total Float ≤ this value are highlighted as critical. MS Project uses 0 by default. Raise to 2–3 days to also flag near-critical tasks as critical.
Dependency types All types Choose between All dependency types (Finish-to-Start, Start-to-Start, Finish-to-Finish, Start-to-Finish) or Finish-to-Start only for a simpler, traditional CPM model.

These settings are saved in the project sidecar file (.custom-props.json) and travel with the project.

Display Preferences (global)

OptionEffect
Show total-float overlay bar on Gantt Draws a light-blue bar on each Gantt task bar proportional to its total float, giving an at-a-glance view of slack directly in the Gantt Chart.
Show Free Float column in Task Sheet Adds a Free Float column to the Task Sheet view.
Show CPM Results panel Makes the CPM Results tab permanently visible without having to click the ribbon button each time.

Display preferences are global (not per-project) and are stored in the Windows registry via QSettings.

Click Restore MS Project Defaults to reset all settings to their defaults (0-day threshold, all dependency types, all display options off).

Sending the CPM Report by Email

The CPM Report split-button in REPORT → Export sends the analysis by email. It requires both an open project and a configured SMTP account.

If no SMTP account is configured, the CPM Report button is disabled and its tooltip directs you to Email Accounts and Email Config in the Report ribbon. See Email Integration for setup instructions.

Two send actions are available via the drop-down arrow (▼):

ActionDescription
Email CPM ReportSends the CPM results table as an HTML email.
Email CPM Report + GanttSends the results table with the full Gantt chart SVG attached.

How the CPM Engine Works

For users who want to understand the calculation:

  1. Forward pass — computes the earliest start and finish for every task, propagating through dependency links.
  2. Backward pass — starting from the project finish date, computes the latest start and finish without delaying the project.
  3. Float calculation — Total Float = Late Finish − Early Finish; Free Float = earliest successor Early Start − own Early Finish.
  4. Completed tasks — tasks at 100% completion are never marked critical, even if their dates would otherwise place them on the critical path.
  5. Summary tasks — a summary task is marked critical if any of its leaf descendants is critical (matching MS Project behaviour).
Tip: After updating task dependencies or durations, the CPM results refresh automatically when you switch back to the CPM Results tab or when a project reload occurs. You do not need to manually trigger a recalculation.
