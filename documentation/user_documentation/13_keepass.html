KeePass Credential Management

Store and retrieve credentials from a local KeePass 4.x database — completely offline, no cloud storage, no plain-text config files.

Home› KeePass Credential Management

What Is KeePass Integration?

Project Offline can unlock a local KeePass 4.x (.kdbx) database and read credentials from it at runtime. This is used in two places:

  • Confluence Calendar Sync — pre-fill the Microsoft Azure AD email and password fields so you only have to complete MFA, not type your password every time.
  • Jira server connections — associate a Jira server with a KeePass entry so the API token / password is never stored in application settings in plain text.

The database is unlocked once per session and locked automatically when the application closes. All decrypted data exists only in RAM.

Offline-first No network round-trip is ever required to read from KeePass. The .kdbx file lives on your local disk; pykeepass decrypts it in-process.

Step 1 — Open or Create a Database

Open Settings → KeePass Configuration… from the menu bar. The dialog presents two modes selected by radio button:

Open existing database

FieldNotes
Database fileBrowse to your .kdbx file (required).
Key fileOptional second factor. Use Browse… to pick an existing file or Generate… to create a new 64-byte binary key file.
Master passwordEnter the database master password. Click Show to reveal it.
Save passwordIf checked, the password is stored base64-encoded in application settings and used for automatic unlock on the next session start.
Unlock buttonTests the credentials without closing the dialog. On success the status line turns green and the Confluence SSO Auto-fill section appears.

Create new database

Choose Create new database to create a brand-new .kdbx file. The file dialog starts in ~/Documents for convenience. Enter a save path (a .kdbx extension is appended automatically if omitted), an optional key file, and a master password. At least one of password or key file is required. The path is converted to an absolute path before the database is written, so the saved path is always fully qualified. Click OK to create the file and unlock it immediately.

Security note Save password stores the master password as base64 in the Windows registry. Base64 is not encryption — it only prevents casual shoulder-surfing. For stronger protection, leave Save password unchecked.

Step 2 — Configure Confluence SSO Auto-fill (optional)

After the database is unlocked, the Confluence SSO Auto-fill section appears at the bottom of the dialog. Select the KeePass entry that holds your Microsoft Azure AD credentials (the same account you use to log in to Confluence).

On the next Project → Sync Confluence Calendar run:

  • If the browser window opens (first run or expired session), the email and password fields are pre-filled automatically from the selected KeePass entry.
  • You only need to complete MFA — no typing of email/password.
  • Subsequent syncs run headlessly; the browser does not open at all.
Entry format: Entries are shown as Group/Title (e.g. Work/Confluence SSO) or just Title for root-group entries. You can also type the entry name directly into the combo box.

Using KeePass for Jira Server Credentials

Open Settings → Jira Servers… and add or edit a server. In the Authentication combo, choose From KeePass entry.

  • The entry combo is pre-populated with all entries from the open database.
  • Click New Entry… to create a new KeePass entry inline — the dialog fills the entry path automatically on return.
  • If the database is not yet configured, a Configure KeePass… button replaces the combo.
  • If the database is configured but locked, an Unlock KeePass… button appears and auto-unlock is attempted first.

Locking & Session Behaviour

TriggerEffect
Application closeDatabase locked automatically; in-memory decrypted data discarded.
Manual lock (Settings menu)Same as application close — session cleared without exiting.
Failed unlock attemptIn-memory reference cleared; user must retry with correct credentials.
Auto-unlock on startIf Save password was checked, the saved password is tried automatically when the first credential lookup is needed.
No idle timer The database is not locked automatically after a period of inactivity in the current release. Lock it manually via Settings → Lock KeePass if you leave your workstation unattended.

Key File Generation

Both the Open existing and Create new pages offer a Generate… button for the key file field. This writes 64 cryptographically random bytes to the chosen path. pykeepass treats a 64-byte binary file as a hex-encoded 32-byte (256-bit) key — each byte of the file represents one nibble of the actual key material — providing full 256-bit key strength with no XML wrapper required.

Keep a backup If you use a key file, losing it means losing access to the database. Store it in a safe location separate from the .kdbx file.

Troubleshooting

SymptomLikely cause & fix
Unlock fails with credentials not correct Wrong password, wrong key file, or both. Check that the key file path matches the one used when the database was created.
Confluence SSO form is not auto-filled The database may be locked. Open Settings → KeePass Configuration and unlock it, then retry the sync.
Entry combo is empty in Jira dialog No entries in the open database, or the database is locked. Unlock first or add entries via the New Entry… button.
KeePass not configured shown in debug dump The keepass/db_path QSettings key is empty. Open the configuration dialog and select a database file.
